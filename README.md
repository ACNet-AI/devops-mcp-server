## Deployment

### Docker Deployment (Recommended)

#### Using docker-compose (Easiest)

```bash
# 1. Start the service
docker-compose up -d

# 2. View logs
docker-compose logs -f

# 3. Stop the service
docker-compose down
```

#### Using Docker commands

```bash
# Build image
docker build -t devops-mcp-server .

# Run container
docker run -d \
  --name devops-mcp-server \
  --env-file .env \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  devops-mcp-server
```

### Local Development

```bash
# Install dependencies
uv sync

# Run server
uv run python server.py
```

### Production Deployment

For production deployment, ensure you:
1. Configure environment variables in `.env`
2. Set up proper logging and monitoring
3. Use `restart: unless-stopped` in docker-compose (already configured)
4. Mount persistent volumes for data

### Health Check

```bash
# Check if container is running
docker ps | grep devops-mcp-server

# View container logs
docker logs devops-mcp-server

# Check service status with docker-compose
docker-compose ps
```
